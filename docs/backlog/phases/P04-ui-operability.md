# P04: UI and Operability

## Goal

Build the web UI, real-time events, and operator action system so humans can monitor and control the autonomous factory.

## Why This Phase Exists

Without a UI, the system is a black box. Operators need visibility and intervention capabilities.

## Exit Criteria

- Dashboard shows live system state
- Task board enables browsing and filtering
- Task detail shows complete timeline and artifacts
- Operator can take all manual actions from the UI
- Real-time updates work for all major state changes

## Risks

UI complexity is high. Prioritize task board and detail views over analytics.
